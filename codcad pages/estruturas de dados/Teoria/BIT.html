Árvore de Indexação Binária (BIT)

Aula de Lucca Siaudzionis

Vamos analisar o problema a seguir. Você tem NN caixas de frutas, númeradas de 11 a NN, e duas possíveis operações.

  • Operação 1: adicionar vv frutas a caixa número kk.
  • Operação 2: responder quantas frutas existem nas caixas de índice entre aa e bb (inclusive).

A solução trivial seria manter um array contendo a quantidade de frutas em cada caixa. Então, para a primeira operação simplesmente se alteraria o valor correspondente no array e para a segunda se iteraria por todas as caixas de índice aa até bb somando-se os valores. A primeira operação seria então feita em O(1)O(1) e a segunda em O(N)O(N). Se quisermos realizar muitas operações do tipo 2, o algoritmo torna-se extremamente ineficiente.

A nova estrutura que abordaremos hoje, Árvore Indexada Binária (mais conhecida como BIT, sua sigla em inglês, ou Árvore de Fenwick), nos permite resolver esse problema de maneira eficiente. Usando-se a BIT (vamos chamar desse jeito, por praticidade), ambas operações são feitas em O(log N)O(log N).

A ideia que permite a BIT fazer ambas operações com essa complexidade consiste de uma divisão de responsabilidade. Em outras palavras, criaremos um array alternativo arvore[]arvore[] tal que arvore[i]arvore[i] não guardará somente a quantidade de frutas na caixa ii, mas a soma de várias caixas consecutivas até a caixa ii. Assim, quando quisermos calcular a soma dos valores das caixas de 11 até kk, achamos os índices certos tal que a soma desses índices no array arvorearvore contenha exatamente os números de 11 a kk sem repetição.

Observação: Nunca vamos calcular diretamente a soma de um intervalo de aa a bb. Iremos sempre calcular a soma do array caixacaixa de 11 até um índice xx. Assim, se quisermos um intervalo de aa a bb, calculamos a soma de 11 até bb e subtraímos da soma de 11 a a−1a−1. Com isso, vamos definir como soma(x)soma(x) a soma caixa[1]+caixa[2]+...+caixa[x]caixa[1]+caixa[2]+...+caixa[x].

Agora, precisamos saber como definir que caixas estarão representadas em arvore[k]arvore[k]. Para isso, vamos definir rr como a posição do primeiro bit 1, da direita pra esquerda, da representação binária do número kk. Assim, arvore[k]arvore[k] irá conter a soma de todas as caixas indo da posição (k−2r+1)(k−2r+1) até kk. Para ilustrar melhor a situação, olhe a seguinte imagem:

Dessa maneira fica mais fácil ver como a divisão é feita. Por exemplo, vemos que soma(13)=arvore[13]+arvore[12]+arvore[8]soma(13)=arvore[13]+arvore[12]+arvore[8]. Porém, se colocarmos tudo em representação binária, a operação fica ainda mais clara: soma(11012)=arvore[11012]+arvore[11002]+arvore[10002]soma(11012)=arvore[11012]+arvore[11002]+arvore[10002]. Note que estamos sempre "apagando" o último bit 1 do número e continuando o algoritmo.

Para fazer o código disso, só precisamos saber como se "apaga" o bit 1 mais à direita. Como vimos na aula de manipulação de bits, o bit menos significante de xx é (xx & −x−x). Assim, temos o seguinte código:


int soma(int x){
	
	int s = 0;
	
	while(x > 0){ // vamos reduzindo x até acabarmos (quando chegamos a zero)
		s += arvore[x]; // adicionamos o pedaço de árvore atual à soma
		x -= (x & -x);  // removemos o bit menos significante
	}
}

Para realizar a operação 2, então, simplesmente calculamos soma(b)−soma(a−1)soma(b)−soma(a−1).

Agora, precisamos pensar em como lidar com a operação 1.

Olhe de novo para a imagem que coloquei anteriormente. Note que, se quisermos atualizar o valor de caixa[5]caixa[5], teremos que atualizar o valor de arvore(5)arvore(5), arvore(6)arvore(6), arvore(8)arvore(8) e arvore(16)arvore(16), pois são todos os valores de arvorearvore que englobam caixa[5]caixa[5]. Se colocarmos os números 55, 66, 88 e 1616, veremos: 001012001012, 001102001102, 010002010002 e 100002100002. O que estamos fazendo, portanto é o contrário do que fizemos na operação 2. Em vez de subtrair o bit menos significante do número, estamos somando-o até chegarmos a um teto (que aqui definimos como 1616).

O código para isso ficaria então muito semelhante ao da função anterior:


void atualiza(int x, int v){ // adicionar v frutas a caixa x
	
	while(x <= N){ // nosso teto, que é quando vamos parar de rodar o algoritmo
		arvore[x] += v; // adicionamos v frutas a arvore[x], como devemos
		x += (x & -x);  // atualizamos o valor de x adicionado ele ao seu bit menos significante
	}
}

A BIT é muito usada para resolver diversos problemas devido a sua complexidade e principalmente a sua facilidade de implementação.
