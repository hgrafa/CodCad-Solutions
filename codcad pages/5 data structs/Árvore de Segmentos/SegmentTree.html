Árvore de Segmentos

Aula de Lucca Siaudzionis

"Succa decidiu virar investidor e enriquecer na bolsa de valores. Para isso, ele possui uma lista de NN opções diferentes (S1,S2,...,SN)(S1,S2,...,SN), cada uma com seu determinado valor. A todo momento, o preço de uma ação pode mudar (subir ou descer). Quando Succa quiser comprar uma ação, ele irá querer saber qual a menor ação dentro de um intervalo [A,B][A,B]. Em outras palavras, ele irá querer saber o valor kk tal que Sk=min(SA,SA+1,...,SB−1,SB)Sk=min(SA,SA+1,...,SB−1,SB). Como tudo ocorre de maneira muito rápida no mercado financeiro, ele pediu que você fizesse um programa para ajudá-lo a enriquecer (e doar parte seu lucro ao CodCad)."

A solução trivial para esse problema é bem imediata. Podemos atualizar o preço de uma ação em O(1)O(1) e achar a ação mais barata em O(N)O(N).

A estrutura que irei apresentar hoje, Árvore de Segmentos, pode fazer cada uma dessas operações em tempo O(log N)O(log N). Porém, o que mais chama a atenção nessa estrutura, como iremos ver, é sua incrível versatilidade. Uma gama enorme de problemas podem ser resolvidos usando-se a mesma ideia básica dessa estrutura.

A ideia por trás da Árvore de Segmentos consiste de, em vez de olhar cada casa do array independentemente, olhamos para vários segmentos contínuos do array. Cada um desses segmentos será um nó em uma árvore binária que iremos construir (daí que surge o nome "Árvore de Segmentos"). Os filhos de um nó que representa os segmento [i,j][i,j] serão os nós que representam os segmentos [i,⌊i+j2⌋][i,⌊i+j2⌋] e [⌊i+j2⌋+1,j][⌊i+j2⌋+1,j]. Assim, estaremos cada segmento ao meio (portanto que i≠ji≠j.

Uma Árvore de Segmentos para um array com N=8N=8 vértices ficaria assim.

Agora, vamos rotular cada um desses nós. Para fazer isso de maneira simples, definiremos o a raiz como tendo o rótulo 11 e seguiremos dessa maneira por todos os outros nós. Então, teremos o seguinte:

Uma observação importante é que os filhos de um nó xx (se existentes) são os nós 2x2x e 2x+12x+1. Da mesma forma, o pai de um nó xx (que não seja a raiz) será o nó ⌊x2⌋⌊x2⌋.

Exercício 2: Como ficaria o desenho da árvore caso N=10N=10?

A versatilidade da Árvore de Segmentos é devido ao fato de podermos salvar o que quiseremos em cada nó. Para resolver nosso problema, salvaremos em um nó que representa o segmento [i,j][i,j] o índice da ação mais barata no intervalo [i,j][i,j]. Agora, se mudarmos o valor da ação kk, só precisamos atualizar o valor dos nós que representam segmentos que contém o valor kk. Você pode observar isso como começar no nó que representa o segmento [k,k][k,k] e ir subindo na árvore até chegar a raiz. O custo desse processo é O(log N)O(log N).

Para implementar a função que faz isso, porém, faremos o caminho contrário. Começaremos a partir da raiz e iremos descendo ao longo da árvore atualizando os vértices necessários. Faremos dessa maneira para padronizar com futuras adaptações da Árvore de Segmentos.

O código da função que atualiza a árvore fica como o seguinte:


// acao[i] representa o preço da ação de índice i
// arvore[i] representa o valor contido no nó i da árvore.
// Ou seja, arvore[i] contém o índice da ação mais barata
// no intervalo representado pelo nó i

// (no) representa o nó que estamos na função recursiva
// o nó que estamos representa o segmento [i, j]
// A função coloca altera o valor da ação de índice (posicao)
// para (novo_valor) e altera a árvore de acordo com o necessário

// sempre teremos que i <= posicao <= j

void atualiza(int no, int i, int j, int posicao, int novo_valor){ 
	
	if(i == j){ // se tivermos i = j, temos i = posicao = j. Logo, estamos no nó mais baixo da ávore
	
		// fazemos então as alterações triviais
		arvore[no] = i;
		acao[posicao] = novo_valor;
	}
	
	else{
		
		int esquerda = 2*no;     // índice do filho da esquerda
		int direita  = 2*no + 1; // índice do filho da direita
		
		// chamamos a função recursiva no nó necessário
		int meio = (i + j)/2;
		
		// se (posicao <= meio), vamos para o nó da esquerda (segmento [i, meio])
		if(posicao <= meio) atualiza(esquerda, i, meio, posicao, novo_valor);
		
		// se não, vamos para o nó da direita (segmento [meio+1, j])
		else atualiza(direita, meio+1, j, posicao, novo_valor);
		
		// agora, pegamos o melhor resultado entre os dois nós
		if( acao[ arvore[esquerda] ] < acao[ arvore[direita] ] ) arvore[no] = arvore[esquerda];
		else arvore[no] = arvore[direita];
	}
	
}

Agora que já vimos como atualizar a árvore, precisamos fazer a outra parte do problema: responder as perguntas do Succa! Para esse passo, também faremos uma função recursiva começando na raiz da árvore para achar a resposta para o intervalo [A,B][A,B]. Quando estivermos em um nó que representa o intervalo [i,j][i,j], realizaremos o seguinte procedimento:

  • se [i,j][i,j] estiver contido em [A,B][A,B] (ou seja, se A≤i≤j≤BA≤i≤j≤B), retornamos a resposta daquele nó. Isso já diz ao nosso programa qual a melhor ação a se comprar dentro do intervalo [i,j][i,j] em O(1)O(1), pois temos essa informação salva.
  • se [i,j][i,j] e [A,B][A,B] forem disjuntos (ou seja, se A>jA>j ou i>Bi>B), nada nesse nó nos interessa. Por definição, retornaremos −1−1.
  • o terceiro caso é dos segmentos se intersectarem. Assim, há valores dentro de [i,j][i,j] que nos interessam e outros que não. Para isso, chamamos a função recursivamente nos filhos do nó que estamos processando.

O código que faz isso pode ser escrito da seguinte maneira:


// queremos saber o índice da ação mais barata dentro do intervalo [A, B]
int consulta(int no, int i, int j, int A, int B){
	
	if(A <= i && j <= B){ // primeiro caso: [i, j] está contido em [A, B]
		return arvore[no];
	}
	
	if(i > B || A > j){ // segundo caso: [i, j] e [A, B] são disjuntos (ou i > j, que significa um nó inválido)
		return -1;
	}
	
	// terceiro caso: chamaremos a função nos dois filhos e retornaremos a melhor resposta
	int esquerda = 2*no;      // índice do filho da esquerda
	int direita  = 2*no + 1;  // índice do filho da direita
	
	int meio = (i + j)/2;
	
	// calculamos a resposta para cada filho
	int resposta_esquerda = consulta(esquerda, i, meio, A, B);
	int resposta_direita  = consulta(direita, meio+1, j, A, B);
	
	// retornamos a melhor resposta
	if(resposta_esquerda == -1) return resposta_direita;  // se o filho da esquerda não possui valor que nos interessa, retornamos o da direita
	if(resposta_direita == -1)  return resposta_esquerda; // se o filho da direita não possui valor que nos interessa, retornamos o da esquerda
	
	// se os dois nos interessam, retornamos o índice de menor custo
	if(acao[resposta_esquerda] < acao[resposta_direita]) return resposta_esquerda;
	else return resposta_direita;
}

Isso já deve ser suficiente para resolver o problema de Succa.

Como disse há pouco, o que faz a Árvore de Segmentos tão poderosa é o fato de ela ser bem versátil. Note que se quiséssemos mudar o código para nos dar o valor da ação mais cara (a que Succa teria que vender), as alterações seriam curtas e intuitivas. Também poderíamos adaptar o código para nos dar a soma dos números dentro de um intervalo, por exemplo. As aplicações dessa estrutura são incontáveis.

Porém, o jeito que abordamos Árvore de Segmentos nessa aula só nos permite alterar um elemento do array por vez. Veremos em breve um complemento a Árvore de Segmentos que nos permite alterar todos os valores em um intervalo [A,B][A,B] em O(log N)O(log N), e então estaremos explorando bem mais o potencial dessa estrutura.
