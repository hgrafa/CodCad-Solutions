Árvore de Indexação Binária (BIT)

Aula de Lucca Siaudzionis

Vamos analisar o problema a seguir. Você tem \(N\) caixas de frutas, númeradas de \(1\) a \(N\), e duas possíveis operações.

  • Operação 1: adicionar \(v\) frutas a caixa número \(k\).
  • Operação 2: responder quantas frutas existem nas caixas de índice entre \(a\) e \(b\) (inclusive).

A solução trivial seria manter um array contendo a quantidade de frutas em cada caixa. Então, para a primeira operação simplesmente se alteraria o valor correspondente no array e para a segunda se iteraria por todas as caixas de índice \(a\) até \(b\) somando-se os valores. A primeira operação seria então feita em \(O(1)\) e a segunda em \(O(N)\). Se quisermos realizar muitas operações do tipo 2, o algoritmo torna-se extremamente ineficiente.

A nova estrutura que abordaremos hoje, Árvore Indexada Binária (mais conhecida como BIT, sua sigla em inglês, ou Árvore de Fenwick), nos permite resolver esse problema de maneira eficiente. Usando-se a BIT (vamos chamar desse jeito, por praticidade), ambas operações são feitas em \(O(log \ N)\).

A ideia que permite a BIT fazer ambas operações com essa complexidade consiste de uma divisão de responsabilidade. Em outras palavras, criaremos um array alternativo \(arvore[]\) tal que \(arvore[i]\) não guardará somente a quantidade de frutas na caixa \(i\), mas a soma de várias caixas consecutivas até a caixa \(i\). Assim, quando quisermos calcular a soma dos valores das caixas de \(1\) até \(k\), achamos os índices certos tal que a soma desses índices no array \(arvore\) contenha exatamente os números de \(1\) a \(k\) sem repetição.

Observação: Nunca vamos calcular diretamente a soma de um intervalo de \(a\) a \(b\). Iremos sempre calcular a soma do array \(caixa\) de \(1\) até um índice \(x\). Assim, se quisermos um intervalo de \(a\) a \(b\), calculamos a soma de \(1\) até \(b\) e subtraímos da soma de \(1\) a \(a-1\). Com isso, vamos definir como \(soma(x)\) a soma \(caixa[1] + caixa[2] + ... + caixa[x]\).

Agora, precisamos saber como definir que caixas estarão representadas em \(arvore[k]\). Para isso, vamos definir \(r\) como a posição do primeiro bit 1, da direita pra esquerda, da representação binária do número \(k\). Assim, \(arvore[k]\) irá conter a soma de todas as caixas indo da posição \((k - 2^r + 1)\) até \(k\). Para ilustrar melhor a situação, olhe a seguinte imagem:

Dessa maneira fica mais fácil ver como a divisão é feita. Por exemplo, vemos que \(soma(13) = arvore[13] + arvore[12] + arvore[8]\). Porém, se colocarmos tudo em representação binária, a operação fica ainda mais clara: \(soma(1101_2) = arvore[1101_2] + arvore[1100_2] + arvore[1000_2]\). Note que estamos sempre "apagando" o último bit 1 do número e continuando o algoritmo.

Para fazer o código disso, só precisamos saber como se "apaga" o bit 1 mais à direita. Como vimos na aula de manipulação de bits, o bit menos significante de \(x\) é (\(x\) & \(-x\)). Assim, temos o seguinte código:


int soma(int x){
	
	int s = 0;
	
	while(x > 0){ // vamos reduzindo x até acabarmos (quando chegamos a zero)
		s += arvore[x]; // adicionamos o pedaço de árvore atual à soma
		x -= (x & -x);  // removemos o bit menos significante
	}
}

Para realizar a operação 2, então, simplesmente calculamos \(soma(b) - soma(a-1)\).

Agora, precisamos pensar em como lidar com a operação 1.

Olhe de novo para a imagem que coloquei anteriormente. Note que, se quisermos atualizar o valor de \(caixa[5]\), teremos que atualizar o valor de \(arvore(5)\), \(arvore(6)\), \(arvore(8)\) e \(arvore(16)\), pois são todos os valores de \(arvore\) que englobam \(caixa[5]\). Se colocarmos os números \(5\), \(6\), \(8\) e \(16\), veremos: \(00101_2\), \(00110_2\), \(01000_2\) e \(10000_2\). O que estamos fazendo, portanto é o contrário do que fizemos na operação 2. Em vez de subtrair o bit menos significante do número, estamos somando-o até chegarmos a um teto (que aqui definimos como \(16\)).

O código para isso ficaria então muito semelhante ao da função anterior:


void atualiza(int x, int v){ // adicionar v frutas a caixa x
	
	while(x <= N){ // nosso teto, que é quando vamos parar de rodar o algoritmo
		arvore[x] += v; // adicionamos v frutas a arvore[x], como devemos
		x += (x & -x);  // atualizamos o valor de x adicionado ele ao seu bit menos significante
	}
}

A BIT é muito usada para resolver diversos problemas devido a sua complexidade e principalmente a sua facilidade de implementação.
